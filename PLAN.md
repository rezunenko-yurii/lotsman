# CodeMap — план реализации

## Этап 0. Каркас
- [x] 0.1 Структура проекта, `pyproject.toml`, пакет `codemap/`
- [x] 0.2 `textutil.py` — оценка токенов (chars/4), разбиение идентификаторов
      (camelCase/snake_case → подтокены), стоп-слова языков

## Этап 1. Индексация
- [x] 1.1 `scanner.py` — обход файлов: `git ls-files` если git-репо, иначе walk
      с чёрным списком директорий; карта расширение→язык; фильтр бинарных/крупных
- [x] 1.2 `store.py` — SQLite-схема: `files`, `symbols`, `idents`; инкрементальный
      upsert по sha256; удаление исчезнувших файлов
- [x] 1.3 `extract.py` — tree-sitter запросы определений для python / js / ts / tsx /
      go / rust / java / c / cpp; сигнатура + строки; regex-fallback; лексический
      сбор идентификаторов (Counter по файлу)
- [x] 1.4 `indexer.py` — связывает 1.1–1.3: инкрементальный проход, статистика

## Этап 2. Ранжирование и карта
- [x] 2.1 `graph.py` — построение графа ссылок (refs→defs, взвешивание: упомянутые
      идентификаторы ×10, приватные ×0.1, слишком общие имена ↓); персонализированный
      PageRank (power iteration, без networkx); распределение ранга по определениям
- [x] 2.2 `repomap.py` — жадный отбор определений под токен-бюджет, рендер
      `file: line: signature`, исключение focus-файлов (они уже в контексте агента)

## Этап 3. Поиск и навигация
- [x] 3.1 `search.py` — BM25 (Okapi) по документам-символам (имя + подтокены +
      сигнатура + путь); бонус за точное совпадение имени
- [x] 3.2 `outline` / `defs` / `refs` — запросы к индексу без пере-ранжирования

## Этап 4. CLI
- [x] 4.1 `cli.py` — подкоманды `index`, `map`, `search`, `outline`, `defs`, `refs`,
      `stats`; флаги `--budget`, `--focus`, `--mention`, `--json`; тайминги в stderr

## Этап 5. Проверка
- [x] 5.1 Юнит-тесты (unittest, без внешних зависимостей): textutil, extract,
      graph/pagerank, bm25, end-to-end на фикстурном мини-репо
- [x] 5.2 Self-index: прогнать CodeMap на самом себе
- [x] 5.3 Полевая проверка на реальном стороннем репозитории (~сотни файлов):
      время индексации, повторной индексации, качество map и search
- [x] 5.4 README.md: установка, команды, **политика использования агентом**
      (сниппет для CLAUDE.md/AGENTS.md)

## Этап 6 (v2). Точные ссылки

      tags-определения — фоллбэк для языков без кастомного запроса
- [x] 6.2 Перезамер качества map на Django (ожидание: generic-имена уходят)

## Этап 7 (v2). Векторный слой и гибридный поиск
- [x] 7.1 `embed.py` — провайдер model2vec (potion-base-8M): локально, без tort/API;

- [x] 7.2 Схема: колонка `vector BLOB` в `symbols` + миграция существующих БД;

- [x] 7.3 `search.py` — режимы `bm25 | vector | hybrid` (RRF, k=60);
